Day 7

Date: 6/12/2017

Intially the design was made with a log in html page with users having to register with our application first and then signing in. After discuessing it with each other we wanted to add the feature of signing in with facebook or google, like modern applications do. After finishing the design and getting the user to register and log in, we decided to remove the registration feature, because we didnt want to deal with user accounts, for efficency we let third party handle users accounts. Therefore for our users to log into our application, they would at least need a facebook or google account.

Using the prototype we had as a foundation to start, our design would initially be like "Whatsupp" or Facebook Messenger. Simplicity would be the aim of the design, simple chat lobby with various features, and a working chat.

Day 10

Date: 21/1/2018

Materialize documentation isnt documented very well, will have to create our own css from some parts, using the javascript provided by the documentation to enable certain features. Discussion was made on where and how the buttons for the "Break,Combine and Delete" was going to be placed and how they would interact with the system. Discuessed how it would change the chat interface, when clicking on "break" or "combine" it would combine the users social media account into one. We thought of this example to explain the funtionality of the buttons.
Example: Hasan logs into Cubik and goes on to services where he logs into facebook, there he picks his conversation with Marin, which is then imported to cubik chat lobby. Hasan then logs into Skype and picks a conversation with Marin he had on skype, where its also imported to cubiks application. Hasan then clicks on the "combine" button, where the two seperate chats combine, where after its ceated, Hasan clicks on the "break" button, which deletes the group.
